SMARTQUALIHOME - System Summary

Overview
SMARTQUALIHOME is a Flask web platform for client qualification, property management, tripping requests, and sales tracking with role-based dashboards for client, agent, and admin users.

Core modules
- app/auth: Authentication, registration, password reset, user onboarding.
- app/client: Client qualification flow, profile updates, and tripping actions.
- app/main: Dashboards, listing operations, PSGC endpoints, admin/agent actions.
- app/ml: Similarity-augmented C5.0 qualification engine.
- app/models.py: SQLAlchemy models for users, profiles, properties, logs, requests, and settings.

Main capabilities
- Role-based access control (client, agent, admin).
- Qualification scoring using DTI + similarity-enhanced C5.0 logic.
- Property lifecycle management (pending, approved, sold, rejected flows).
- Tripping scheduling and request management.
- Full property detail request workflow (pending, approved, rejected).
- Activity logs and in-dashboard notification handling.
- Configurable qualification and security settings through system_config.

Data and storage
- Development default: SQLite at instance/smartqualihome.db.
- Production option: MySQL via app/config.py and migrations/schema.sql.
- Uploads: instance/uploads.

Startup behavior
On app startup, the system:
1) Creates/opens required instance paths.
2) Initializes Flask extensions and blueprints.
3) Attempts C5.0 warm-up from training_data when at least 10 rows are available.
4) Falls back safely to rule-based assessment if model warm-up fails or data is insufficient.
