{# DTI Threshold Rules #}
DTI Qualification Thresholds
Debt-to-Income ratio bands for client eligibility classification
Debt-to-Income ratio bands are used by the C5.0 classification engine to determine client eligibility. Lower DTI means less debt pressure and a stronger chance of passing the screening.
Example: 35% and below = Qualified
Example: 36% to 42% = Conditionally Qualified
Example: above 42% = Not Qualified

DTI ≤ this → Qualified
DTI ≤ this → Conditionally Qualified
DTI > Conditional max → Not Qualified (derived)
{# Algorithm & Risk Settings #}
Algorithm & Risk Settings
C5.0 decision tree and similarity measure interaction controls
These settings control how strict the qualification engine behaves. The confidence threshold decides when the tree prediction is strong enough to use on its own. If the case is borderline, the similarity measure checks historical buyer patterns to help finalize the result. The minimum tenure and minimum income values act as extra risk filters before the final status is shown.
Example: confidence 72% or higher = use the tree result directly
Example: confidence below 72% = treat as borderline and compare historical similarity
Example: 6 months tenure and PHP 15,000 income are the default minimums

Employment Stability Scores
Stability weights per employment type used by the C5.0 feature encoder
These values are stability weights, not approval rules. The C5.0 feature encoder converts employment type into a score so the engine can compare how stable each applicant looks. Higher scores mean the system sees that employment type as more stable for lending purposes.
Scale: minimum is 1, maximum is 5
